Meter Monitor

Elektr, suv va gaz monitoringi uchun yagona platforma. ESP32 qurilmalar sensorlardan ma'lumot o'qib, HTTP orqali backendga yuboradi. Dashboard real vaqtda grafik, alert va boshqaruv imkoniyatlari beradi.

⚡ Elektr (TE71/TE73 DLMS) 💧 Suv 🔥 Gaz ESP32 · FastAPI · React
🖥️

Desktop App

TE71/TE73 hisoblagich bilan DLMS/COSEM orqali to'g'ridan-to'g'ri ishlash

📡

ESP32 Firmware

RS-485, bosim va flow sensorlari uchun universal firmware

⚡

Real-time Dashboard

WebSocket orqali jonli grafik va alertlar

🔔

Alert Tizimi

Threshold qoidalar, eskalatsiya va Telegram/webhook bildirishnomalar

🔄

OTA Updates

ESP32 firmwareni masofadan yangilash va batch rollout

💾

Backup / Restore

Admin panel orqali ma'lumot eksport va import

Arxitektura

Tizim to'rtta mustaqil komponentdan iborat:

Sensor / Hisoblagich │ RS-485 (DLMS/COSEM) yoki analog bosim sensori ▼ ESP32 ────── universal firmware (elektr / suv / gaz) │ HTTP POST /api/readings │ X-Device-Token: <token> ▼ FastAPI Backend ──── PostgreSQL + SQLAlchemy async │ WebSocket /ws ▼ React Dashboard ──── Vite + PWA + React Query Desktop App ────── PyQt6 · DLMS/COSEM · RS-485 │ To'g'ridan-to'g'ri hisoblagichga ulanadi (backendsiz) │ Rele boshqarish, registrlarni o'qish, vaqt sinxron
ℹ️

Backend MQTT ishlatmaydi. ESP32 bilan asosiy aloqa HTTP orqali. WebSocket faqat frontend uchun real-time event layer.

Ma'lumot oqimi

  1. ESP32 RS-485/sensor orqali o'qiydi → POST /api/readings
  2. Backend utility_type bo'yicha ajratib PostgreSQL ga yozadi
  3. WebSocket barcha ulangan clientlarga reading event yuboradi
  4. React Query cache yangilanadi → UI yangilanadi

Desktop App

TE71 (1-fazali) va TE73 (3-fazali) hisoblagichlar bilan DLMS/COSEM protokoli orqali bevosita ishlash uchun PyQt6 dasturi. Backendga ulanish shart emas.

O'rnatish

BASH
cd desktop
pip install -r requirements.txt
python main.py

Ulanish parametrlari

HisoblagichBaudParityStop bitsAuth
TE71 (1-fazali)9600N1Reader / Public / Manager
TE73 (3-fazali)4800N1Reader / Public / Manager

Auth rejimlari

RejimClient IDImkoniyat
Public16Asosiy parametrlarni o'qish (parolsiz)
Reader1Barcha registrlar + rele boshqarish (HLS5)
Manager2Vaqt sinxron, tarif, parol o'zgartirish (LOW)

Asosiy funksiyalar

  • Dashboard — Kuchlanish, tok, quvvat, chastota, energiya real-time
  • Rele boshqarish — Remote reconnect / disconnect tasdiqlash dialogi bilan
  • Registrlar — Barcha OBIS kodlarni o'qish, custom OBIS so'rovi
  • Sozlamalar — Vaqt sinxron, tarif rejasi, parol o'zgartirish
  • Log — HDLC frame log, hodisalar jurnali
  • ESP32 Flash — Firmware yuklash va serial monitor

ESP32 Firmware

Universal firmware uchta utility type ni qo'llab-quvvatlaydi: elektr (TE71/TE73 RS-485 DLMS), suv (bosim sensori) va gaz (bosim/flow sensori).

Build va yuklash

BASH
# PlatformIO CLI
pio run                          # Build
pio run -t upload                # Yukla
pio device monitor               # Serial monitor

# Yoki Desktop App orqali → "ESP32 ga firmware yuklash"

Konfiguratsiya (platformio.ini)

INI
[env:esp32]
platform = espressif32
board    = esp32dev
build_flags =
    '-DDEFAULT_SERVER_URL="http://192.168.1.100"'
    '-DDEFAULT_WIFI_SSID="MyWiFi"'
    '-DDEFAULT_WIFI_PASS="password"'
    '-DDEVICE_TOKEN="T30gwzZJ6YTvQeLRMCZyTi-GBAYogsQV"'

Ishga tushish tartibi

  1. ESP32 yoqilganda WiFi ga ulanadi
  2. GET /api/device-config/{device_id} — o'z konfiguratsiyasini so'raydi
  3. Asosiy sensor/hisoblagichdan o'lchov o'qiydi
  4. POST /api/readings — backendga yuboradi
  5. OTA check qiladi, yangi firmware bo'lsa yuklab o'rnatadi
ℹ️

Asosiy server ishlamasa, ESP32 backup serverlar ro'yxatidan qayta urinadi. Serverlar soni konfig orqali beriladi.

API — Autentifikatsiya

POST /api/auth/login

Username va parol bilan kirish. JWT access token qaytaradi.

JSON
// Request
{ "username": "admin", "password": "Admin1234" }

// Response
{ "access_token": "eyJ...", "token_type": "bearer" }

GET /api/auth/users admin

Foydalanuvchilar ro'yxati. q, role, status, sort_by, sort_order, limit, offset parametrlari qo'llab-quvvatlanadi.

Production Deployment

Minimal talablar

  • PostgreSQL 15+ (SQLite faqat development)
  • RAM: 512 MB+, CPU: 1 core+
  • HTTPS — reverse proxy (nginx / Caddy)
  • Persistent volume: /app/data (firmware va backup fayllar)

Docker Compose (production)

BASH
docker compose -f docker-compose.yml up -d --build
docker compose exec backend alembic upgrade head

Migratsiya (yangi versiya)

BASH
docker compose exec backend alembic upgrade head

Checklist

  • ✅ SECRET_KEY o'zgartirilgan (openssl rand -hex 32)
  • ✅ DEVICE_API_TOKEN o'zgartirilgan
  • ✅ Admin parol o'zgartirilgan
  • ✅ APP_ENV=production
  • ✅ HTTPS sozlangan
  • ✅ PostgreSQL persistent volume
  • ✅ Backup yo'nalishi persistent volume
  • ✅ alembic upgrade head ishga tushirilgan
✅

GitHub Actions CI/CD orqali backend va frontend avtomatik deploy qilinadi. main branchga push = production deploy.
