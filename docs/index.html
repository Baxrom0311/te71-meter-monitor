Meter Monitor

Elektr, suv va gaz hisoblagichlarini bitta tizimda kuzatish uchun platforma. Server serverga o'rnatiladi. ESP32 qurilmalar sensorlardan o'qib, serverga yuboradi. Desktop App hisoblagichga to'g'ridan-to'g'ri ulanib diagnostika va boshqaruv qiladi.

Nimalar kerak

🖥️

Server yoki kompyuter

Docker o'rnatilgan Linux server. RAM: 512MB+. Internet: ixtiyoriy (lokal ham ishlaydi).

📡

ESP32 platasi

Standard ESP32-WROOM yoki ESP32-S3. RS-485 uchun MAX3485 konvertor modul.

⚡

Hisoblagich (TE71/TE73)

Toshelectroapparat elektr hisoblagichlari. RS-485 interfeysi kerak.

🔌

USB-RS485 adapter

Desktop App uchun. CH340 yoki FTDI chipli adapter. Kompyuterga ulanadi.

💡

Server va ESP32 bir xil tarmoqda bo'lishi shart emas. ESP32 internet orqali ham ulanishi mumkin. Desktop App esa faqat lokal USB-RS485 orqali ishlaydi — serverga ulanmaydi.

Qanday ishlaydi

TE71/TE73 Hisoblagich │ RS-485 kabel ▼ ESP32 ── WiFi orqali serverga │ HTTP POST /api/readings · har 30 soniyada ▼ Backend (server) ── PostgreSQL ma'lumotlar bazasi │ WebSocket real-time yangilanish ▼ Dashboard (brauzer) ── grafik, alert, boshqaruv ───────────────────────────────────────────── Desktop App ── to'g'ridan-to'g'ri, serverga ulanmaydi │ USB-RS485 · DLMS/COSEM protokol ▼ TE71/TE73 Hisoblagich ── diagnostika, rele, registrlar

Ikki yo'l mustaqil ishlaydi. ESP32 + backend — uzluksiz monitoring. Desktop App — texnik xodim vazifasida hisoblagich bilan bevosita ishlash.

① Backend — serverga o'rnatish

Backend + dashboard + ma'lumotlar bazasi hammasi bitta Docker Compose faylida. Boshqa hech narsa kerak emas.

1

Kodni yuklab oling

BASH
git clone https://github.com/your-org/meter-monitor.git
cd meter-monitor
2

Sozlamalarni to'ldiring

BASH
cp .env.example .env

Keyin .env faylini oching va quyidagilarni o'zgartiring:

SECRET_KEY
JWT tokenlar uchun maxfiy kalit. Ixtiyoriy uzun matn.
Generatsiya: openssl rand -hex 32
DEVICE_API_TOKEN
ESP32 qurilmalar shu token bilan ma'lumot yuboradi.
Misol: T30gwzZJ6YTvQeLRMCZyTi-GBAYogsQV
BOOTSTRAP_ADMIN_PASSWORD
Dashboard admin paroli. Birinchi tushirishda yaratiladi.
Default: Admin1234 (albatta o'zgartiring)
3

Ishga tushiring

BASH
docker compose up -d --build

Birinchi marta 3-5 daqiqa ketadi (yuklab olish va build). Keyingi safar 10 soniya.

4

Ishga tushganini tekshiring

BASH
curl http://localhost:8000/health
# {"status": "ok"} — tayyor!

Dashboard: http://SERVER_IP:8000 — brauzerda oching.

✅

Kirish: admin / .env dagi parol

Server manzilini aniqlash

ESP32 va boshqalar ulansa kerak bo'lgan manzilingizni bilib oling:

BASH
# Linux serverda:
hostname -I | awk '{print $1}'
# Misol natija: 192.168.1.50

# Keyin bu manzil barcha joyda ishlatiladi:
# http://192.168.1.50:8000

③ Desktop App — hisoblagichga ulanish

USB-RS485 adapterni kompyuterga ulang. Adapterni hisoblagichning RS-485 portiga ulang (A+ va B– simlari).

2

Dasturni ishga tushiring

BASH
cd desktop
pip install -r requirements.txt   # bir marta
python main.py
3

④ Dashboard — ma'lumotlarni kuzatish

1

Kirish

Brauzerda http://SERVER_IP:8000 ga o'ting. Login: admin, parol: .env dagi qiymat.

2

Binoni yarating

Buildings → "Yangi bino". Nom va manzil kiriting. Qurilmalar binolarga biriktiriladi va grafik shu bino bo'yicha ajratib ko'rsatiladi.

3

Qurilmani biriktiring

Devices sahifasida yangi (yashil) qurilmani toping. Ustiga bosib, binoga biriktiring va nom bering.

4

Grafik ko'rinadi

Dashboard da qurilma holati (online/offline), oxirgi o'lchov qiymatlari va soatlik grafik ko'rinadi. O'lchov kelar ekan, grafik avtomatik yangilanadi.

✅

Hammasi tayyor. Qurilmalar ma'lumot yuborib turadi, dashboard real vaqtda yangilanadi.

ESP32 Firmware xizmati

Qurilma ID qayerda ko'rinadi

Har bir ESP32 ning MAC manzili uning ID si bo'ladi (masalan: F42DC96D5820). Birinchi o'lchov yuborganda dashboard da Devices sahifasida paydo bo'ladi.

Serverga ulanmayapti — nima qilish kerak

  • WiFi SSID va parol to'g'rimi? (Serial monitor da tekshiring)
  • Server manzili to'g'rimi? http:// bilan boshlanishi kerak
  • Device Token .env dagi DEVICE_API_TOKEN bilan bir xilmi?
  • Server ishlayaptimi? curl http://SERVER_IP:8000/health

Firmware versiyasini masofadan yangilash (OTA)

Dashboard → OTA sahifasiga o'ting:

  • 1
    Yangi .bin faylni "Upload firmware" orqali yuklab oling

    Firmware ma'lumotlarini to'ldiring (versiya, utility type)
  • 3
    "Yangi batch" yarating va qurilmalarni tanlang

OTA — masofadan firmware yangilash

ESP32 larni server orqali yangilash. Bitta qurilma yoki bir nechta — barchasi bir vaqtda.

QadamNima qilish kerak
1Dashboard → OTA → "Firmware yuklash" → .bin faylni tanlang
2Versiya, utility type (elektr/suv/gaz) to'ldiring
3"Yangi batch" → qurilmalarni belgilang
4"Process" — ESP32 lar o'zi yuklab o'rnatadi (1-5 daqiqa)
5Status: Pending → In Progress → Completed
ℹ️

OTA vaqtida ESP32 ma'lumot yuborishni to'xtatadi. O'rnatib bo'lgach avtomatik qayta ishga tushadi va davom etadi.

Muhit o'zgaruvchilari (.env)

Muammolar va yechimlar

Backend ishga tushmayapti

BASH
docker compose logs backend        # xatolik sababini ko'rish
docker compose ps                  # container holati

Ko'p uchraydigan sabab: .env fayli yo'q yoki DATABASE_URL noto'g'ri.

ESP32 serverga ulanmayapti

  • Serial monitor da WiFi holati tekshiring
  • Server IP to'g'rimi? http:// bilan boshlanishi kerak
  • DEVICE_API_TOKEN .env dagi qiymat bilan bir xilmi?
  • Firewall 8000-portni yopdimi? sudo ufw allow 8000

Desktop App COM portni ko'rmayapti

  • USB-RS485 adapter drayveri o'rnatildimi? (CH340 yoki FTDI)
  • "Yangilash" tugmasini bosing
  • macOS da: ls /dev/tty.usb* bilan portni toping
  • Linux da: sudo usermod -aG dialout $USER — keyin qayta kirish

Hisoblagichga ulanmayapti

  • TE71 → 9600 baud, TE73 → 4800 baud. To'g'ri turdagi tanlangmi?
  • RS-485 simlar: A+ qizil, B– qora (yoki teskarisi — sinab ko'ring)
  • Hisoblagich quvvati bormi?
  • "Qo'lda sozlash" rejimida baud va parity ni qo'lda kiriting

Dashboard da grafik bo'sh

Bu normal. Qurilma ma'lumot yubora boshlagach grafik to'ladi. Yoki qurilma binoga biriktirilmagandir — Devices sahifasida tekshiring.
